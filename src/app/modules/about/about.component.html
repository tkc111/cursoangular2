Ejercicio 1

  • Escribe una clase en TypeScript que represente una nota tipo Evernote. Debe tener un título, una fecha, un autor, y un contenido. El contenido no debe ser necesariamente un string, pudiendo venir representado por otro tipo de objetos.
  • Para probarla, escribe un archivo typescript que cree varias notas a modo de ejemplo y las muestre en un listado web.

Ejercicio 2

  • Escribe una pequeña aplicación Angular2 que liste un conjunto de notas tipo Evernote. Añade un buscador para poder buscar las notas por cualquier parte del título o del contenido.
  • Supón que las notas provienen de un servicio externo mapeado a través de un servicio de Angular, para ello haz que, como en clase, se devuelva un simple mock estático.

Ejercicio 3

Sobre la aplicación construida para el Ejercicio 2, añade la siguiente funcionalidad:
  • El usuario debe poder ver una nota en detalle.
  • El usuario debe poder acceder desde la vista de detalle a un formulario para su edición. Por ahora solo atenderemos a campos textuales y fechas.
  • Utiliza pipes para controlar el formato de la fecha

Ejercicio 4

Sobre la aplicación construida para el Ejercicio 3, añade la siguiente funcionalidad:
  • Cambia el “layout” para que se adapte a lo siguiente:
    • Crea una cabecera que permanezca durante toda la navegación.
    • La cabecera debe tener dos opciones: Notas, y Configuración.
    • Bajo ella un área de trabajo que desplegará el contenido según la opción seleccionada en la cabecera
      • Notas debe desplegar el listado de notas
      • Configuración debe desplegar opciones de configuración de dos tipos:
        • Perfil de usuario: nombre, email, password. (Por ahora datos falsos o guardamos en localstorage)
        • Configuración app: No es esencial que exista funcionalidad aquí
      • Tanto el área de notas como la de configuración deben tener un módulo y enrutamiento propios.
    • Utiliza el servidor de ejemplo para guardar y rescatar datos utilizando un api REST.
